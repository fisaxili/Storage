# Информационная система управления базой данных «Склад»

Учебный проект — настольное приложение для автоматизации складского учёта. Программа предназначена для работы с данными о деталях, поставщиках, складах и поставках. Проект разработан в рамках учебной практики и демонстрирует применение трёхслойной архитектуры, технологий Windows Forms и Microsoft Access.

## Стек

* .NET 8 (Windows Forms)
* Microsoft Access (.accdb) через OleDb (Microsoft.ACE.OLEDB.12.0)
* xUnit для модульного тестирования
* Microsoft.Extensions.DependencyInjection для внедрения зависимостей (DI)

## Структура проекта

```
Storage/
├── DAL/                      # Слой доступа к данным
│   ├── DbManager.cs          # Подключение к БД и работа с пользователями
│   ├── DataRepository.cs     # CRUD-операции над таблицами
│   ├── IDbManager.cs         # Интерфейс для тестирования
│   └── Queries/             # SQL-запросы
│
├── BLL/                      # Бизнес-логика
│   ├── AuthService.cs        # Авторизация и защита от брутфорса
│   ├── DataService.cs        # Проверка прав доступа и CRUD
│   ├── UserSession.cs        # Информация о текущем пользователе
│   └── Secure/
│       └── PasswordHasher.cs # Хеширование SHA-256
│
├── UI/                       # Формы Windows Forms
│   ├── MainMenu.cs          # Главное меню
│   ├── Login/               # Авторизация
│   ├── Register/            # Регистрация
│   ├── Profile/             # Личный кабинет
│   ├── Database/            # Работа с таблицами БД
│   ├── Reports/             # Отчёты и экспорт CSV
│   └── About/               # О программе
│
├── Models/                   # Модели данных
├── Storage.Tests/            # Юнит-тесты xUnit
├── StorageDB.accdb             # Файл базы данных
└── App.config                # Строка подключения
```

## Структура базы данных

### Детали

| Поле                | Назначение                      |
| ------------------- | ------------------------------- |
| Код_Детали (PK)     | Уникальный идентификатор детали |
| Наименование_Детали | Название детали                 |
| Цена_Детали         | Стоимость детали                |

### Поставщики

| Поле                | Назначение                          |
| ------------------- | ----------------------------------- |
| Код_Поставщика (PK) | Уникальный идентификатор поставщика |
| Название_Поставщика | Наименование организации            |
| Адрес_Поставщика    | Адрес                               |
| Страна              | Страна                              |
| Индекс              | Почтовый индекс                     |
| Банковский_Счёт     | Расчётный счёт                      |

### Склады

| Поле              | Назначение              |
| ----------------- | ----------------------- |
| Номер_Склада (PK) | Идентификатор склада    |
| ФИО_Кладовщика    | Ответственный сотрудник |
| Телефон           | Контактный телефон      |

### Поставка

| Поле                | Назначение             |
| ------------------- | ---------------------- |
| Код_Поставки (PK)   | Идентификатор поставки |
| Код_Поставщика (FK) | Поставщик              |
| Код_Детали (FK)     | Деталь                 |
| Номер_Склада (FK)   | Склад                  |
| Количество_Деталей  | Количество товара      |

### Пользователи

| Поле                  | Назначение                 |
| --------------------- | -------------------------- |
| Код_Пользователя (PK) | Идентификатор пользователя |
| Логин                 | Учётная запись             |
| Пароль                | SHA-256 хеш пароля         |
| Роль                  | Роль пользователя          |

### Связи

* Поставщики 1 — ∞ Поставка
* Детали 1 — ∞ Поставка
* Склады 1 — ∞ Поставка

## Запуск проекта

Установить Microsoft Access Database Engine 2016 Redistributable (x64).

Файл `Storage.accdb` должен находиться в корневой директории проекта и автоматически копироваться в папку сборки.

Сборка проекта:

```bash
dotnet build
```

Запуск приложения:

```bash
dotnet run --project Storage.csproj
```

## Тестирование

Запуск тестов:

```bash
dotnet test
```

Тесты покрывают:

* PasswordHasher;
* UserSession;
* AuthService;
* проверку ролей пользователей;
* механизмы авторизации;
* защиту от неверного ввода.

## Безопасность

* Параметризованные SQL-запросы
* Хеширование паролей алгоритмом SHA-256
* Защита от SQL-инъекций
* Защита от брутфорса (3 ошибки входа → блокировка на 30 минут)
* Разграничение прав доступа по ролям
* Проверка прав на уровне бизнес-логики (DataService)

## Архитектура

Приложение построено по трёхслойной архитектуре:

```
UI (Windows Forms)
        ↓
BLL (AuthService, DataService)
        ↓
DAL (DataRepository, DbManager)
        ↓
Microsoft Access
```
Такое разделение обеспечивает масштабируемость, удобство сопровождения и возможность дальнейшего расширения функциональности системы.
